# ATM System

Высокоуровневая программная система для управления банкоматом, состоящая из распределенного **Backend-сервиса (REST API)** и **интерактивного CLI-клиента**. Проект спроектирован с упором на соблюдение принципов **чистой архитектуры** и эффективную работу с реляционными базами данных.

## 🏗 Архитектурные особенности

Проект реализует **Луковую архитектуру**, что обеспечивает полную независимость бизнес-логики от внешних инструментов (БД, интерфейсов, сторонних библиотек).

- **Rich Domain Model:** Основная логика инкапсулирована внутри доменных сущностей, обеспечивая согласованность данных.
- **Persistence Layer:** Реализован на низкоуровневом драйвере **Npgsql**. Это обеспечивает максимальную производительность и полный контроль над запросами к базе данных без использования тяжелых ORM.
- **Migrations:** Схема базы данных управляется через **FluentMigrator**, что позволяет версионировать структуру таблиц с помощью чистого SQL.
- **Dependency Injection:** Все модули системы связаны через **MSDI**, реализуя принцип инверсии зависимостей.

## 🚀 Функциональные возможности

### Система сессий и авторизация
- **Два уровня доступа:** 
  - **Пользовательский:** Авторизация по номеру счета и PIN-коду.
  - **Административный:** Доступ к системным функциям по мастер-паролю.
- **Session Management:** После логина система выдает уникальный ключ сессии (Guid), который используется для валидации всех последующих операций.

### Банковские операции
- Просмотр текущего баланса в реальном времени.
- Пополнение счета и снятие наличных с проверкой остатка.
- Автоматическая фиксация каждой операции в истории транзакций.

### Консольный клиент
- Построен на базе **Spectre.Console**, предоставляя пользователю интерактивные меню, таблицы и формы ввода.
- Взаимодействие с сервером реализовано через типизированные клиенты **Refit** с ручной обработкой HTTP-статусов без использования исключений.

## 🛠 Технологический стек

- **Backend:** .NET 10, ASP.NET Core.
- **Database:** PostgreSQL, Npgsql, FluentMigrator.
- **Infrastructure:** Docker, IOptions.
- **Client:** Spectre.Console, Refit.
- **API Documentation:** OpenAPI (Swagger/Scalar UI).

## 📂 Структура решения

Система разделена на два основных блока в директории `src`:
- **`service/`** — серверная часть, содержащая слои домена, приложения (Use Cases) и инфраструктуры.
- **`console-client/`** — клиентское приложение, реализующее слой представления для конечного пользователя.
